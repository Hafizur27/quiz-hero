1. what is the difference between map(), filter(), forEach() & find() methods ?

map() filter() forEach() find()
map method returns a new array with the same length as the original array, where each element is transformed by the callback function passed as an argument. The original array is not modified. filter method returns a new array with some or all of the elements of the original array, depending on whether they pass a test specified by the callback function. The original array is not modified. forEach method executes a provided function once for each array element, but it doesn't return anything. It's mainly used for performing side effects on each element of the array. find method returns the first element in the array that satisfies a specified test provided by the callback function. If no element satisfies the test, undefined is returned.
    Example:
  • const Array = [1, 2, 3];
  • const newArray = Array.map(element => element * 2);
  • console.log(newArray);
  • newArray: [2, 4, 6]
  • console.log(Array);
  • Array: [1, 2, 3]
    Example:
  • const Array = [1, 2, 4];
  • const newArray = Array.filter(element => element % 2 === 0);
  • console.log(newArray);
  • newArray: [2, 4]
  • console.log(Array);
  • Array: [1, 2, 4]
    Example:
  • const Array = [1, 2];
  • const newArray = Array.forEach(element => element % 2 === 0);
  • firstElement: 1
  • secondElement: 2
  • console.log(Array);
  • Array: [1, 2,]
    Example:
  • const Array = [1, 2, 3];
  • const newValue = Array.find(element => element === 2);
  • console.log(newValue);
  • newValue: 2
  • console.log(Array);
  • Array: [1, 2, 3]

2. What is the difference between local storage, session storage & cookies ?

Local Storage Session Storage Cookies
Local storage is a type of web storage that allows web applications to store and access data in a user's web browser. It provides a way for web applications to store persistent data locally on the user's device, even after the user has closed the browser window. The data stored in local storage remains available until it is explicitly cleared by the user or the web application that stored it. Local storage is typically used to store settings, preferences, or other application-specific data that needs to persist between browser sessions. Session storage is similar to local storage, but the data stored in it is only available for the duration of a single browser session. When the user closes the browser window, the data stored in session storage is automatically cleared. Session storage is typically used to store temporary data, such as a user's shopping cart contents or other transient data that is only needed during the current session. Cookies are small text files that are stored on a user's device by a web browser. They are typically used to store information about the user's preferences, login information, and other data that needs to persist between browser sessions. Cookies can be set to expire after a specific time period or when the user closes the browser window. They can also be set to be accessible only to specific web pages or domains. Cookies are often used for tracking user behavior and delivering personalized content.
