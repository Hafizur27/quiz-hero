1. what is the difference between map(),
filter(), forEach() & find() methods ?

map() :- map method returns a new array with the same length as the original array, where each element is transformed by the callback function passed as an argument. The original array is not modified.
Example:
const Array = [1, 2, 3];
const newArray = Array.map(element => element * 2);
console.log(newArray);
newArray: [2, 4, 6]

filter() :- filter method returns a new array with some or all of the elements of the original array, depending on whether they pass a test specified by the callback function. The original array is not modified.
Example:
const Array = [1, 2, 4];
const newArray = Array.filter(element => element % 2 === 0);
console.log(newArray);
newArray: [2, 4]

forEach() :- forEach method executes a provided function once for each array element, but it doesn't return anything. It's mainly used for performing side effects on each element of the array.
Example:
const Array = [1, 2];
Array.forEach(element => console.log(element));
firstElement: 1
secondElement: 2

find() :- find method returns the first element in the array that satisfies a specified test provided by the callback function. If no element satisfies the test, undefined is returned.
Example:
const Array = [1, 2, 3];
const newValue = Array.find(element => element === 2);
console.log(newValue);
newValue: 2

2. What is the difference between
local storage, session storage & cookies ?

Local Storage :- localStorage is a property that allows JavaScript sites and apps to save key-value pairs in a web browser with no expiration date. This means the data stored in the browser will persist even after the browser window is closed. The client can only read local storage. There is no transfer of data to the server.The storage capacity of local storage is 5MB/10MB.As it is not session-based, it must be deleted via javascript or manually.

session Storage :- Session storage allows you to store data in the browser depending on the system memory and the data stored in the browser until the browser is closed. In other words, closing the browser will clear all the data stored in session storage. The client can only read session storage.There is no transfer of data to the server. The storage capacity of session storage is 5MB. It's session-based and works per window or tab. This means that data is stored only for the duration of a session.

Cookies :- A cookie is an amount of information that persists between a server-side and a client-side. A web browser stores this information at the time of browsing. A cookie contains the information as a string generally in the form of a name-value pair separated by semi-colons. Both clients and servers can read and write the cookies. Data transfer to the server is exist. The storage capacity of Cookies is 4KB. Cookies expire based on the setting and working per tab and window.

3. What is event loop and how does work it?

The event loop is a mechanism used in many programming languages, including JavaScript, to handle asynchronous operations and non-blocking I/O. It is a continuous process that enables an application to respond to multiple events simultaneously without blocking the main thread or freezing the user interface.
In JavaScript, the event loop operates by continuously monitoring two queues: the call stack and the task queue. When the call stack is empty, the event loop checks the task queue for any pending tasks that can be executed. If there are any pending tasks, the event loop moves them from the task queue to the call stack, and the JavaScript engine executes them one by one.
If a pending task involves an asynchronous operation, such as a network request or a timer, it is scheduled to run in the background by the runtime environment, and the JavaScript engine continues executing other tasks from the call stack. When the asynchronous operation is completed, the runtime environment adds the corresponding callback function to the task queue.
The event loop processes the task queue and executes the callback function, updating the state of the application or triggering new asynchronous operations. The event loop also has a microtask queue, which is a separate queue that holds small and fast-running tasks, such as promises or process.nextTick() callbacks. The microtask queue is processed before the task queue, ensuring that these high-priority tasks are executed as soon as possible.
